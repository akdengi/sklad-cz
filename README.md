# Товароучёт + Честный Знак

Автор: Александр Казанцев

Система учёта товаров с интеграцией Честный Знак: маркировка, генерация КМ (DataMatrix), управление складами, выбытие, этикетки.

Работает на **Windows** и **Linux**.

Лицензия: [MIT](LICENSE)

## Требования

- **Python 3.11+**
- **Ghostscript** — для генерации DataMatrix (treepoem)
- **КриптоПро CSP** — **обязателен** для автоматической проверки статусов маркировки через API ЧЗ (без него придётся проверять статусы вручную в ЛК Честный Знак)
- **Внешний сканер штрихкодов** — рекомендуется (USB, режим Keyboard Wedge)

### Проверка DataMatrix

**Настоятельно рекомендуется** проверять напечатанные DataMatrix-коды через мобильное приложение **«ЧЗ.Бизнес»** (App Store / Google Play):
- После печати каждой этикетки — убедитесь, что код считывается корректно
- В карточке товара — сканируйте для проверки GTIN и серийного номера
- При поступлении от поставщика — проверяйте все коды

Это защитит от проблем при проверке на кассе, маркетплейсе и при подаче отчётов.

### Windows

- **Visual C++ Redistributable 2013** — для нативных модулей Python: https://learn.microsoft.com/ru-ru/cpp/windows/latest-supported-vc-redist
- **Ghostscript** — скачать: https://github.com/ArtifexSoftware/ghostpdl-downloads/releases (gs10071w64.exe)
- **КриптоПро CSP** — (опционально) для ЭЦП: https://www.cryptopro.ru/products/csp

### Linux

```bash
sudo apt install ghostscript python3-dev gcc
```

Для ЭЦП: установить КриптоПро CSP Linux и модуль `pycades`.

## Установка

### 1. Установить Ghostscript

**Windows:** скачать с https://github.com/ArtifexSoftware/ghostpdl-downloads/releases, при установке отметить "Add to PATH".

**Linux:**
```bash
sudo apt install ghostscript
```

### 2. Создать виртуальное окружение и установить зависимости

**Windows:**
```
setup.bat
```

**Linux / Mac:**
```bash
chmod +x setup.sh run.sh
./setup.sh
```

Скрипт автоматически:
- Создаёт папку `venv/` с виртуальным окружением
- Обновляет pip
- Устанавливает все зависимости из `requirements.txt`

Зависимости подбираются по платформе:
- Windows: `pywin32` (COM-интерфейс КриптоПро)
- Linux: `pycades` (интеграция с КриптоПро)

### 3. Запуск

**Windows:**
```
start.bat
```

**Linux / Mac:**
```bash
./run.sh
```

Откроется в браузере: http://127.0.0.1:5000

> При первом запуске `start.bat`/`run.sh` без venv покажет инструкцию по установке.

## Функционал

### Склады
- Два типа: **физический** и **виртуальный** (маркетплейс)
- Статистика по каждому SKU на складе (в наличии / продано)
- Переименование и смена типа
- Удаление с подтверждением

### Продажа
- Быстрое оформление продажи по коду ЧЗ или EAN-13
- Автоматический перенос кода с физического склада на виртуальный перед продажей
- Выбор маркетплейса (Ozon, Яндекс Маркет и т.д.)
- Товары без маркировки — продажа без отчёта о выбытии в ЧЗ

### Товары (SKU)
- Управление товарами с GTIN, артикулом, партией
- Два типа: **с маркировкой (ЧЗ)** и **без маркировки**
- Для товаров без маркировки GTIN-14 не обязателен
- Прогресс маркировки

### Остатки
- Таблица единиц с фильтрами и сортировкой
- Перемещение между складами
- Просмотр детальной информации и статуса ЧЗ
- Подсветка цветом: розовый — оффлайн-валидация не пройдена, оранжевый — проблемный статус ЧЗ
- Колонки **«Статус»** и **«Статус в ЧЗ»**
- **Массовая проверка ЧЗ** — выделите товары чекбоксами и нажмите «Проверить статус в ЧЗ»

### Продано
- Список проданных товаров с суммой
- Колонки **«Статус ЧЗ»** и **«Вывод из оборота»**
- Кнопка **«Вернуть на склад»** для товаров с подтверждённым отчётом
- **Массовая проверка ЧЗ** — выделите товары чекбоксами и нажмите «Проверить статус в ЧЗ»

### Вывод из оборота
- Отчёты о выбытии в Честный Знак
- Колонки **«Статус отчёта»** и **«Статус в ЧЗ»**
- Два статуса отчёта: **«Не начато»** и **«Подтверждено ЧЗ»**
- **Автоподтверждение** — при статусе ЧЗ = Выбыл отчёт автоматически подтверждается
- Кнопка **«Вернуть в оборот»** для подтверждённых отчётов
- **Массовая проверка ЧЗ** — выделите товары чекбоксами и нажмите «Проверить статус в ЧЗ»

### Сканирование
- Массовое сканирование кодов ЧЗ

### Импорт / Экспорт
- Импорт кодов из CSV/PDF
- Экспорт в JSON, CSV

### Этикетки
- Генерация PDF, ZPL, TSPL этикеток
- Для товаров с маркировкой: DataMatrix + текст + EAN-13
- Для товаров без маркировки: текст + EAN-13 (без DataMatrix)

### Данные
- Экспорт/импорт JSON
- Очистка БД
- Бэкапы с ротацией
- Настройки Честный Знак

### Удалённая синхронизация
- Загрузка/скачивание базы данных через SSH (push/pull)
- Автоматические бэкапы при синхронизации

### Проверка статусов ЧЗ
- Массовая проверка статусов маркировки
- Проверка отдельного товара по кнопке
- Кэширование токена авторизации
- Сброс статуса при ошибке (КМ не найден в ЧЗ)

### Оффлайн-валидация КМ
- Проверка структуры кодов маркировки без обращения к API ЧЗ
- Автоматическая валидация при импорте, сканировании и создании единиц
- Подсветка ошибок в таблице Остатков

## Удалённая синхронизация

Позволяет работать с одной базой данных на нескольких компьютерах через SSH.

### Настройка

1. Перейти во вкладке **Данные** → секция **Удалённая синхронизация**
2. Ввести IP сервера, логин, пароль и путь к директории Sklad на сервере
3. Нажать **Сохранить настройки**

### Использование

- **Загрузить на сервер** — копирует БД и настройки на сервер. Предыдущая БД сохраняется как `.bak`.
- **Скачать с сервера** — скачивает БД и настройки с сервера. Локальная БД автоматически бэкапируется перед заменой.
- После синхронизации требуется перезапуск приложения.

### Через командную строку

Синхронизация тоже работает через venv. На Windows перед выполнением нужно активировать окружение:

```bash
venv\Scripts\activate
python sync.py push    # загрузить базу на сервер
python sync.py pull    # скачать базу с сервера
python sync.py status  # статус синхронизации
```

На Linux:
```bash
source venv/bin/activate
python sync.py push
python sync.py pull
python sync.py status
```

### Бэкапы

- Локально хранится до 10 бэкапов в `backups/`
- При каждом pull создаётся локальный бэкап предыдущей версии

## Проверка статусов Честный Знак

Позволяет получать актуальный статус маркировки товаров из системы Честный Знак.

### Требования

- **КриптоПро CSP** — установленный на компьютере
- **Сертификат ЭЦП** — действующий, в хранилище Windows/Linux или на токене (RuToken и т.д.)
- Для подписи:
  - Windows: `pywin32` (уже в requirements.txt)
  - Linux: `pycades` (уже в requirements.txt)

### Настройка

1. Вставить токен (если сертификат на нём)
2. Нажать **«Найти»** в настройках ЧЗ — система покажет все доступные сертификаты
3. Выбрать нужный сертификат из списка (отпечаток заполнится автоматически)
4. **«Сохранить»** → **«Проверить авторизацию»**
5. При подписи КриптоПро запросит PIN-код (если токен)

### Использование

- **Обновить статусы всех товаров** — проверяет все коды ЧЗ в базе пакетами по 100 штук. Токен кэшируется, при 401 автоматически обновляется. Коды без ответа от ЧЗ получают пустой статус.
- **Оффлайн-валидация всех кодов** — проверяет структуру кодов (GTIN, серийный номер, ключ, криптохвост) без обращения к API. Результат сохраняется и отображается цветом в Остатках.
- **В карточке товара** — кнопка «Проверить» / «Обновить» рядом со статусом ЧЗ. Если код не найден — статус сбрасывается, отображается «КМ не найден в ЧЗ».

## Решение проблем с авторизацией ЧЗ

### Ошибка "Подпись невалидна. Код ошибки: 4"
`csptest` производит PKCS#7, а API требует **CAdES-BES**. Нужно:
- Установить `csptestf` (входит в состав КриптоПро CSP)

### Ошибка 403 "Отсутствует доступ к ресурсу"
- Проверьте, что сертификат добавлен как пользователь API на markirovka.crpt.ru
- Используйте sandbox для тестирования: `https://markirovka.sandbox.crptech.ru/api/v3/true-api`

### Диагностика
Нажмите «Диагностика» в настройках ЧЗ — проверяет каждый шаг авторизации.

## Формат кода ЧЗ

Код маркировки (КМ) хранится с FNC1 (`\xe8`) в начале и GS (`\x1d`) перед AI 91/92:

```
FNC1 01 04600000000000 21 серийныйномер GS 91 ключ GS 92 криптохвост
```

При поиске используется только часть до AI 91 (префикс).

Генерация DataMatrix: treepoem с `parsefnc=True`, FNC1 передаётся как `^FNC1` (BWIPP синтаксис).
