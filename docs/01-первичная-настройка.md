# 1. Первичная настройка

## Системные требования

- **Python 3.11+**
- **Ghostscript** — для генерации штрихкодов DataMatrix
- **КриптоПро CSP** — **обязателен** для автоматической проверки статусов маркировки через API ЧЗ. Без него нельзя получить и обновить статусы программно — придётся проверять каждый код вручную через личный кабинет Честный Знак
- **Внешний сканер штрихкодов** — рекомендуется для удобной работы с кодами маркировки

### Windows

1. Установите **Visual C++ Redistributable 2013**: https://learn.microsoft.com/ru-ru/cpp/windows/latest-supported-vc-redist
2. Установите **Ghostscript**: скачайте с https://github.com/ArtifexSoftware/ghostpdl-downloads/releases, при установке отметьте «Add to PATH»
3. Установите **КриптоПро CSP**: https://www.cryptopro.ru/products/csp — нужен для автоматической проверки статусов ЧЗ

### Linux

```bash
sudo apt install ghostscript python3-dev gcc
```

Для ЭЦП: установите КриптоПро CSP Linux и модуль `pycades`.

## Установка

### Шаг 1 — Зависимости Python

```bash
pip install -r requirements.txt
```

Зависимости подбираются автоматически:
- Windows: `pywin32` (COM-интерфейс КриптоПро)
- Linux: `pycades` (интеграция с КриптоПро)

### Шаг 2 — Настройка окружения

Скопируйте `.env.example` в `.env`:

```
CZ_API_URL=https://markirovka.crpt.ru/api/v3/true-api
```

### Шаг 3 — Запуск

```bash
python run.py
```

Откройте в браузере: http://127.0.0.1:5000

## Все параметры настроек

Вкладка **«Настройка»** содержит три блока настроек.

### Блок 1: Честный Знак — статусы

| Параметр | Обязательно | Описание |
|----------|-------------|----------|
| **URL API** | Да | Адрес API ЧЗ. По умолчанию: `https://markirovka.crpt.ru/api/v3/true-api`. Для тестов используйте sandbox: `https://markirovka.sandbox.crptech.ru/api/v3/true-api` |
| **ИНН организации** | Да | ИНН 10 или 12 цифр. Используется при авторизации в API ЧЗ |
| **Отпечаток сертификата (SHA-1)** | Да | Отпечаток сертификата ЭЦП. Нажмите **«Найти»** для автопоиска сертификатов с приватным ключом в хранилище Windows/Linux |
| **PIN токена** | Да (если токен) | PIN-код от токена/карты (RuToken, JaCarta и т.д.). Запрашивается при подписании |

### Блок 2: Адрес вывода из оборота по умолчанию

| Параметр | Описание |
|----------|----------|
| **Адрес** | Адрес места выбытия товара. Подставляется автоматически при каждой продаже. Пример: `г. Город, ул. Пример, д. 1` |
| **FIAS ID** | UUID адреса из ФИАС. Получить можно на https://fias.nalog.ru. Пример: `f311252f-e8a7-46cf-8125-36927c45b9b7` |

### Блок 3: Бэкапы и синхронизация

| Параметр | Описание |
|----------|----------|
| **Ротация бэкапов** | Включает автоматическое создание бэкапа при каждом запуске приложения |
| **IP сервера** | IP-адрес удалённого сервера для SSH-синхронизации |
| **Логин** | SSH-логин (по умолчанию: `root`) |
| **Пароль** | SSH-пароль |
| **Удалённый путь** | Путь к директории Sklad на сервере |

## Работа с криптоподписью (ЭЦП)

### Что нужно для работы

1. **Сертификат ЭЦП** — действующий, выпущенный Accredited CA (УЦ ФНС, Калуга Астрал и др.)
2. **КриптоПро CSP** — установленный на компьютере
3. **Токен/карта** — если сертификат на носителе (опционально, может быть в хранилище Windows)

### Поиск сертификата

1. Вставьте токен (если используется)
2. В настройках ЧЗ нажмите **«Найти»**
3. Система покажет все сертификаты с приватным ключом из хранилищ:
   - **Local Machine** (Windows) / **Local Machine** (Linux)
   - **Current User** (Windows) / **Current User** (Linux)
4. Выберите нужный сертификат — отпечаток заполнится автоматически

### Как работает подпись

При проверке статусов ЧЗ приложение:

1. Запрашивает ключ аутентификации у API ЧЗ (`GET /auth/key`)
2. Получает `uuid` и `data` (случайные данные для подписи)
3. **Подписывает `data`** с помощью КриптоПро CSP:
   - Windows: COM-объект `CAdESCOM.CadesSignedData` (CAdES-BES)
   - Linux: библиотека `pycades` (CADESCOM_CADES_BES)
4. Отправляет подпись на `POST /auth/simpleSignIn`
5. Получает bearer-токен для дальнейших запросов

### Типы подписи

| Тип | Описание |
|-----|----------|
| **CAdES-BES** | Требуется API ЧЗ. Стандартный формат |
| **PKCS#7** | Производит `csptest` — **не подходит** для API ЧЗ |

### Решение проблем с подписью

| Проблема | Решение |
|----------|---------|
| «Подпись невалидна. Код ошибки: 4» | Установите `csptestf` (входит в КриптоПро CSP). Стандартный `csptest` производит PKCS#7, а API требует CAdES-BES |
| «Отсутствует доступ к ресурсу» (403) | Сертификат не добавлен как пользователь API на markirovka.crpt.ru. Зарегистрируйте сертификат |
| Сертификат не найден | Проверьте, что токен подключён и КриптоПро CSP установлен. Нажмите «Диагностика» |
| PIN-код неверен | Проверьте PIN. Обычно это 6 цифр. При блокировке токена — обратитесь в УЦ |

### Диагностика

Нажмите **«Диагностика»** в настройках ЧЗ — система проверит пошагово:

1. **Сертификат** — найден ли сертификат с приватным ключом
2. **PIN** — задан ли PIN-код
3. **Подпись** — можно ли подписать тестовые данные
4. **Авторизация** — получение ключа и прохождение `simpleSignIn`

Результаты отображаются для каждого шага с деталями ошибок.

## Бэкапы

### Автоматический бэкап

При включённой **ротации бэкапов** при каждом запуске приложения создаётся резервная копия БД:

```
backups/backup_20260705_153000.db
```

Хранится до **8 последних бэкапов** (настраивается через `MAX_BACKUPS` в `.env`). Самые старые удаляются автоматически.

### Ручной бэкап

1. Перейдите в **«Настройка»** → секция **«Резервное копирование»**
2. Нажмите **«Создать бэкап»**
3. Бэкап появится в списке

### Восстановление из бэкапа

1. Выберите бэкап из списка
2. Нажмите **«Восстановить»**
3. Подтвердите
4. **Перезапустите приложение**

### Экспорт/импорт JSON

- **«Экспорт JSON»** — полный дамп базы (склады + товары + единицы)
- **«Импорт JSON»** — **ВНИМАНИЕ:** заменяет ВСЕ данные в базе. Перед импортом создайте бэкап!

## Удалённая синхронизация (SSH)

Позволяет работать с одной базой данных на нескольких компьютерах.

### Настройка

1. Перейдите в **«Настройка»** → **«Удалённая синхронизация»**
2. Заполните: IP сервера, логин, пароль, удалённый путь
3. Сохраните

### Загрузка на сервер (Push)

- Копирует локальную БД на сервер
- Перед заменой автоматически создаётся бэкап на сервере
- Также синхронизируется `settings.json`

### Скачивание с сервера (Pull)

- Скачивает БД с сервера
- Локальная БД бэкапируется перед заменой
- Также синхронизируется `settings.json`

### Через командную строку

```bash
python sync.py push    # загрузить на сервер
python sync.py pull    # скачать с сервера
python sync.py status  # статус синхронизации
```

## Проверка данных от ЧЗ

### Как убедиться, что API ЧЗ работает

1. Нажмите **«Проверить авторизацию»** — система попытается получить токен
2. При успехе: «Авторизация прошла успешно. Токен получен.»
3. При ошибке: нажмите **«Диагностика»** для деталей

### Проверка одного товара

1. Откройте карточку единицы (вкладка «Остатки» → «Изменить»)
2. Нажмите **«Проверить»** рядом с КМ кодом
3. Система запросит статус из API ЧЗ и отобразит результат

### Массовая проверка

1. В настройках нажмите **«Обновить статусы всех товаров»**
2. Дождитесь завершения (коды проверяются пакетами по 900)
3. Статусы единиц обновятся в таблице

### Что показывает проверка

| Поле | Описание |
|------|----------|
| **cz_status** | Текущий статус в ЧЗ: EMITTED, APPLIED, INTRODUCED, RETIRED и т.д. |
| **cz_check_date** | Дата последней проверки |

### Отладка

Если проверка не работает:
1. Используйте **«Отладка запроса»** — введите ID единицы, нажмите «Отладка запроса»
2. Система покажет: raw-код, длину, hex-представление, результат запроса к API

## Оборудование

### Внешний сканер штрихкодов (рекомендуется)

Для удобной работы рекомендуется использовать **внешний USB-сканер штрихкодов** (захват клавиатуры). Это значительно ускоряет:
- Сканирование КМ при маркировке
- Быструю продажу по коду
- Поиск товаров

Сканер должен быть в режиме **Keyboard Wedge** (эмуляция клавиатуры) — код автоматически вводится в активное поле ввода.

Рекомендуемые модели: Honeywell, Zebra, Datalogic — любые с поддержкой DataMatrix.

### Проверка DataMatrix через ЧЗ.Бизнес

**Рекомендуется всегда проверять напечатанные DataMatrix-коды** через мобильное приложение **«ЧЗ.Бизнес»** (доступно в App Store и Google Play):

- **После печати этикетки** — убедитесь, что код считывается и содержит правильный GTIN и серийный номер
- **В карточке товара** — сканируйте код для проверки корректности нанесения
- **При поступлении товара** — проверяйте коды от поставщика

Это защитит от ошибок при генерации этикеток и проблем при проверке на кассе или маркетплейсе.
