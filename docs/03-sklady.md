# 3. Создание складов (физические и виртуальные)

## Типы складов

Приложение поддерживает два типа складов:

| Тип | Назначение | Примеры |
|-----|------------|---------|
| **Физический** | реальное место хранения товаров | Основной склад, Магазин, Склад-2 |
| **Виртуальный** | маркетплейс или канал продаж | Озон, Яндекс Маркет, Wildberries |

## Создание склада

1. Перейдите во вкладку **«Склады»**
2. Нажмите **«Новый склад»**
3. Заполните:
   - **Название** — имя склада (должно быть уникальным)
   - **Тип** — «Физический» или «Виртуальный (маркетплейс)»
   - **Цвет** — цвет карточки для визуального разделения
4. Нажмите **«Сохранить»**

## Склады по умолчанию

При первом запуске создаются три склада:
- **Склад** (физический) — основное место хранения
- **Озон** (виртуальный)
- **Яндекс Маркет** (виртуальный)

## Как работают склады

### Физические склады
- Единицы товара хранятся на физическом складе до продажи
- При сканировании КМ код привязывается к единице на выбранном физическом складе
- Можно перемещать единицы между физическими складами

### Виртуальные склады
- Используются при продаже через маркетплейс
- При продаже товара КМ код перемещается с физического склада на виртуальный
- На каждом виртуальном складе не может быть более одной активной (непроданной) единицы одного SKU

### Пример workflow

1. Товар поступил → создайте единицу на **физическом складе**
2. Товар промаркирован → КМ нанесён
3. Продажа через Озон → товар перемещается на склад **«Озон»** (виртуальный)
4. Статус меняется на «Продан»

## Статистика

На карточке каждого склада отображается:
- Количество единиц в наличии
- Количество проданных
- Разбивка по товарам (SKU)
