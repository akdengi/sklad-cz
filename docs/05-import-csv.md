# 5. Импорт кодов маркировки из CSV или TXT

## Формат файла

Система принимает файлы:
- **CSV** / **TSV** — разделители запятая или табуляция
- **TXT** — один код на строку

Каждая строка должна содержать код маркировки длинной более 20 символов с цифрами.

## Импорт через интерфейс

1. Перейдите во вкладку **«Импорт»**
2. Выберите:
   - **Товар (SKU)** — к какому товару привязать коды
   - **Склад** — на какой склад определить единицы
   - **Статус** — статус присваиваемый единицам
3. Нажмите **«Выберите файл»** и загрузите CSV/TXT
4. Нажмите **«Импортировать CSV»**
5. Результаты отобразятся в журнале:
   - **Добавлено** — новые единицы созданы
   - **Привязано** — коды присвоены существующим пустым единицам
   - **Дубликаты** — коды уже существуют в базе
   - **Ошибки** — некорректные коды

### Проверки при импорте

При массовом импорте система выполняет следующие проверки для каждого кода:

1. **Дубликат** — проверка, не существует ли уже такой код в базе
2. **Структура КМ** — проверка GTIN (AI 01), серийного номера (AI 21), ключа (AI 91), криптохвоста (AI 92)
3. **Соответствие GTIN** — извлечение GTIN из КМ и проверка, есть ли SKU с таким GTIN в базе
4. **Статус ЧЗ онлайн** — проверка статуса маркировки через API Честный Знак

### Окно ошибок

Если при импорте обнаружены ошибки, открывается модальное окно с таблицей, содержащей:
- **Код КМ** — код с ошибкой
- **Причина** — тип ошибки (дубликат, нет GTIN, ошибка структуры, нет SKU для GTIN, ошибка проверки ЧЗ и т.д.)
- **Подробности** — описание ошибки

Нажмите **«Скачать CSV»** для экспорта таблицы ошибок.

## Импорт из PDF

Система может извлекать коды маркировки из PDF-файлов:

1. Перейдите во вкладку **«Импорт»**
2. Выберите SKU, склад и статус
3. Загрузите PDF-файл
4. Система автоматически найдёт все GS1 DataMatrix паттерны

## Экспорт данных

### Экспорт единиц (CSV)

В настройках нажмите **«Экспорт единиц (CSV)»** — скачается CSV со всеми единицами и их полями.

### Экспорт для отчёта выбытия (CSV)

В настройках нажмите **«Экспорт выбытия (CSV)»** — специальный формат для подачи отчёта в ЧЗ:
- Полный КМ код
- Код для отчёта (всё до GS+91)
- Формат Озон (с экранированием `\u001d`)

### Экспорт JSON

Полный дамп базы данных в JSON — для резервного копирования или переноса.

## Осторожно

- **Импорт JSON** заменяет ВСЕ данные в базе
- **Очистка базы** удаляет всё безвозвратно
- Перед этими операциями создайте резервную копию
