# 15. API Честный Знак — техническое описание

## Обзор

Приложение интегрируется с **CrPT API** (Центр развития перспективных технологий) — государственной системой маркировки товаров «Честный Знак». Интеграция реализована в модуле `app/cz_api.py` и обеспечивает:

- Аутентификацию по сертификату ЭЦП (CAdES-BES)
- Получение JWT-токена для доступа к API
- Проверку статусов кодов маркировки (КМ)
- Диагностику подключения и сертификатов

## Базовый URL API

```
Продакшн: https://markirovka.crpt.ru/api/v3/true-api
Песочница: https://markirovka.sandbox.crptech.ru/api/v3/true-api
```

Настраивается через переменную окружения `CZ_API_URL` или в настройках приложения.

## Аутентификация

### Принцип работы

Аутентификация использует **挑战-ответ механизм** с подписью сертификата ЭЦП:

```
┌─────────────────────────────────────────────────────────────────┐
│  Клиент                          Сервер CrPT                   │
│                                                                 │
│  ─── GET /auth/key ──────────────────────────────────────────►  │
│  ◄── { uuid, data } ─────────────────────────────────────────  │
│                                                                 │
│  Подпись data приватным ключем сертификата                      │
│  (CAdES-BES формат)                                            │
│                                                                 │
│  ─── POST /auth/simpleSignIn ────────────────────────────────►  │
│  { uuid, data: signature, unitedToken: true, inn }              │
│  ◄── { token } ──────────────────────────────────────────────  │
│                                                                 │
│  Токен используется как Bearer в Authorization                  │
└─────────────────────────────────────────────────────────────────┘
```

### Шаг 1: Получение ключа аутентификации

**Запрос:**
```
GET /auth/key
Accept: application/json
```

**Ответ:**
```json
{
  "uuid": "уникальный-идентификатор-сессии",
  "data": "данные-для-подписи"
}
```

### Шаг 2: Формирование подписи

Данные `data` подписываются приватным ключем сертификата в формате **CAdES-BES** (не PKCS#7!).

**Реализация:**

| Платформа | Модуль | Ключевые вызовы |
|-----------|--------|-----------------|
| Windows | `win32com` + CAdESCOM | `CAdESCOM.CPSigner`, `CAdESCOM.CadesSignedData.SignCades()` |
| Linux | `pycades` | `pycades.SignedData.SignCades()` с `CADESCOM_CADES_BES` |

**Критично:** API принимает **только CAdES-BES** подпись. Если используется `csptest` (стандартный CSP), он выдает PKCS#7 — это вызовет ошибку 403 «Подпись невалидна». Необходимо:
- Установить `csptestf` (входит в КриптоПро CSP), либо
- Использовать CAdESCOM напрямую (как реализовано в приложении)

### Шаг 3: Получение токена

**Запрос:**
```
POST /auth/simpleSignIn
Content-Type: application/json
Accept: application/json

{
  "uuid": "из-шага-1",
  "data": "CAdES-BES-подпись",
  "unitedToken": true,
  "inn": "7700000000"  // опционально
}
```

**Ответ (успех):**
```json
{
  "token": "eyJhbGciOiJSUzI1NiIs..."
}
```

**Ответ (ошибка):**
```json
{
  "error_message": "Подпись невалидна"
}
```

### Кеширование токена

Токен кешируется в глобальной переменной `_uuid_token` и переиспользуется до истечения. При получении ошибки 401 токен автоматически сбрасывается и запрашивается заново.

## API endpoints

### Авторизация

| Метод | URL | Описание |
|-------|-----|----------|
| `GET` | `/auth/key` | Получение ключа для подписи |
| `POST` | `/auth/simpleSignIn` | Аутентификация, получение JWT-токена |

### Информация о КМ

| Метод | URL | Описание |
|-------|-----|----------|
| `POST` | `/cises/info` | Получение информации о кодах маркировки |

**Запрос:**
```json
{
  "Authorization": "Bearer <token>",
  "Content-Type": "application/json"
}

[
  "010460000000000021SERIAL_NUMBER",
  "010460000000000121ANOTHER_SERIAL"
]
```

**Ограничение:** максимум 1000 кодов за запрос. При массовой проверке коды разбиваются на пакеты по 900.

**Ответ:**
```json
[
  {
    "cis": "010460000000000021SERIAL_NUMBER",
    "cisInfo": {
      "status": "INTRODUCED",
      "productGroup": "Обувь",
      "cis": "010460000000000021SERIAL_NUMBER"
    },
    "errorCode": "0",
    "errorMessage": ""
  }
]
```

## Статусы кодов маркировки

| Статус CrPT | Описание | Статус в приложении | Действие |
|-------------|----------|---------------------|----------|
| `EMITTED` | Код выпущен системой ЧЗ | 1 — Эмитирован | Нанести код на товар |
| `APPLIED` | Код нанесён на товар | 2 — Нанесён | Подать отчёт о нанесении |
| `INTRODUCED` | Товар введён в оборот | 3 — В обороте | Готов к продаже |
| `RETIRED` | Товар выбыл из оборота | 5 — Выбыл | Отчёт подан |
| `WITHDRAWN` | Товар выведен из оборота | 5 — Выбыл | Отчёт подан |
| `WRITTEN_OFF` | Товар списан | 5 — Выбыл | Отчёт подан |

**Важно:** статус в приложении обновляется **только в сторону увеличения** (никогда не понижается).

---

## Массовая проверка статусов

### Алгоритм

1. Получение всех единиц с кодами ЧЗ из БД
2. Очистка кодов (удаление FNC1, GS, обрезка по AI 91)
3. Разбиение на пакеты по 900 кодов
4. Для каждого пакета:
   - Запрос `POST /cises/info`
   - При ошибке 401 — сброс токена и повтор
   - Обновление статусов в БД
5. Коды без ответа от ЧЗ получают пустой статус

### API проверки

```bash
# Запуск массовой проверки
POST /api/cz/check-all

# Проверка статуса выполнения
GET /api/cz/check-status

# Проверка одного товара
POST /api/cz/check-single
{ "unit_id": 123 }
```

## Работа с сертификатами

### Получение списка сертификатов

```bash
GET /api/cz/list-certs
```

**Ответ:**
```json
{
  "ok": true,
  "certs": [
    {
      "thumbprint": "ABC123...",
      "subject": "CN=Иванов Иван, O=ООО Ромашка",
      "issuer": "CN=УЦ ФНС России",
      "has_private_key": true,
      "store": "Local Machine"
    }
  ]
}
```

**Фильтрация:**
- Только сертификаты с приватным ключем (`has_private_key: true`)
- Опциональная фильтрация по ИНН из настроек

### Хранилища сертификатов

| Хранилище | Windows | Linux |
|-----------|---------|-------|
| Текущий пользователь | `CADESCOM_CURRENT_USER_STORE` | `pycades.CADESCOM_CURRENT_USER_STORE` |
| Локальный компьютер | `CADESCOM_LOCAL_MACHINE_STORE` | `pycades.CADESCOM_LOCAL_MACHINE_STORE` |

## Диагностика

API `/api/cz/diagnose` выполняет пошаговую проверку:

```bash
POST /api/cz/diagnose
```

**Шаги проверки:**

| Шаг | Проверка | Ожидаемый результат |
|-----|----------|---------------------|
| 1. `certificate` | Наличие отпечатка сертификата | Thumbprint установлен |
| 2. `pin` | Наличие PIN-кода | PIN задан |
| 3. `signing` | Тестовая подпись | Подпись сформирована |
| 4. `auth_key` | Запрос ключа аутентификации | UUID и data получены |
| 5. `simpleSignIn` | Полная авторизация | Токен получен |

**Пример ответа:**
```json
{
  "ok": true,
  "steps": [
    { "step": "certificate", "ok": true, "detail": "Thumbprint: ABC123DEF456..." },
    { "step": "pin", "ok": true, "detail": "Set" },
    { "step": "signing", "ok": true, "detail": "csptest OK (344 chars)" },
    { "step": "auth_key", "ok": true, "detail": "uuid=abc123..., data=xyz789..." },
    { "step": "simpleSignIn", "ok": true, "detail": "Token received (512 chars)" }
  ]
}
```

### Тест авторизации

```bash
POST /api/cz/test-auth
{ "thumbprint": "ABC123..." }
```

Сбрасывает кеш токена и выполняет полную процедуру аутентификации.

## Обработка ошибок

### Ошибки аутентификации

| Код | Причина | Решение |
|-----|---------|---------|
| 403 | Подпись невалидна | Установить `csptestf` или использовать CAdESCOM |
| 403 | Отсутствует доступ | Зарегистрировать сертификат как API-пользователь на markirovka.crpt.ru |
| 401 | Токен истек | Автоматический сброс и повторный запрос |

### Ошибки запросов

| Код | Причина | Решение |
|-----|---------|---------|
| 400 | Невалидный запрос | Проверить формат данных |
| 404 | КМ не найден | Код не зарегистрирован в ЧЗ |
| 429 | Превышен лимит запросов | Подождать и повторить |

## Требования

### Обязательные компоненты

- **КриптоПро CSP** — установленный на компьютере
- **Сертификат ЭЦП** — действующий, с приватным ключем
- **Доступ к API** — сертификат зарегистрирован как API-пользователь

### Зависимости Python

| Платформа | Пакет | Назначение |
|-----------|-------|------------|
| Windows | `pywin32` | COM-интерфейс с CAdESCOM |
| Linux | `pycades` | Интеграция с КриптоПро |

## Примеры использования

### Проверка статуса одного товара

```python
from app.cz_api import get_uuid_token, get_cis_info

token = get_uuid_token(thumbprint="ABC123...")
result = get_cis_info(token, ["010460000000000021SERIAL123"])
print(result[0]["cisInfo"]["status"])  # "INTRODUCED"
```

### Массовая проверка

```python
from app.cz_api import check_cz_status

codes = ["010460000000000021SERIAL001", "010460000000000021SERIAL002"]
result = check_cz_status(codes)
for entry in result["results"]:
    print(f"{entry['cis']}: {entry['cisInfo']['status']}")
```

### Диагностика через API

```bash
# Проверка всех шагов
curl -X POST http://127.0.0.1:5000/api/cz/diagnose

# Тест авторизации
curl -X POST http://127.0.0.1:5000/api/cz/test-auth \
  -H "Content-Type: application/json" \
```
