# 10. Процесс продажи товара

## Общая схема

```
Товар на складе
    ↓
Продажа через кассу / маркетплейс
    ↓
Оформление продажи в приложении (корзина)
    ↓
(Для ЧЗ) Подача отчёта о выводе из оборота через ЛК ЧЗ
```

## Быстрая продажа (корзина)

Основной способ продажи. Позволяет оформить заказ из нескольких позиций за одно действие.

### Шаг 1. Заполнение шапки заказа

1. Перейдите во вкладку **«Продажа»**
2. Введите **номер заказа** (например, номер заказа маркетплейса)
3. Выберите **склад списания** (виртуальный склад маркетплейса)
4. Укажите **дату продажи** (по умолчанию — сегодня)
5. Выберите **вид документа** (по умолчанию — «Прочее»): Прочее, УПД, Товарная накладная, Акт приёма-передачи, Кассовый чек

### Шаг 2. Добавление товаров в корзину

В поле **«Код ЧЗ / Артикул / EAN-13»** вводите товары по одному:

- **С маркировкой**: отсканируйте или вставьте **КМ-код** Честного Знака
- **Без маркировки**: введите **EAN-13**, артикул или название товара

При вводе кода автоматически выполняется поиск товара и отображается информация:
- Единица #ID, SKU, артикул
- Склад, статус
- Статус в ЧЗ (для маркированных)
- КМ для Маркетплейсов (для маркированных)

Для каждой позиции укажите **цену** и нажмите **«Добавить»** (или Enter).

### Шаг 3. Просмотр корзины

Все добавленные позиции отображаются в таблице:
- **#** — порядковый номер
- **Товар** — SKU и артикул
- **Код ЧЗ / КМ** — полный КМ для маркетплейсов (нажмите для копирования)
- **Цена** — редактируется inline
- **Удалить** — кнопка удаления позиции

Итого отображается в заголовке корзины и на кнопке продажи.

### Скачать КМ в CSV

Под таблицей корзины есть кнопка **«Скачать КМ в csv»**. Нажмите её, чтобы скачать CSV-файл со всеми КМ из корзины в формате «КМ для ввода в оборот и вывода из него» (код до первого GS-символа). Файл содержит только коды КМ, по одному на строку, без заголовков.

### Передача данных через УПД (ЭДО)

Если вид документа chosen **«УПД»**, перед отправкой заказа:

1. Добавьте все товары в корзину
2. Нажмите **«Скачать КМ в csv»** — файл со всеми КМ скачается
3. Передайте скачанные КМ через **ЭДО вашей компании** или **ЭДО Лайт** (встроенное решение Честного Знака)
4. После передачи данных через ЭДО нажмите **«Продать»**

> Важно: скачайте КМ **до** нажатия кнопки «Продать», так как после продажи статус товаров изменится.

### Шаг 4. Продажа

Нажмите **«Продать всё (N поз., X ₽)»**:
- Каждый товар обрабатывается отдельно
- Маркированные товары — через эндпоинт `/api/units/quick-sell`
- Товары без маркировки — через эндпоинт `/api/units/sell-no-marking`
- Результат: количество проданных / ошибки
- Корзина очищается, номер заказа и склад сохраняются для следующего заказа

### Особенности

- Одинаковый **номер заказа** и **склад** для всех позиций
- Разная **цена** для каждой позиции
- В **«Проданных»** и **«Вывод из оборота»** все позиции — отдельные строки
- Поиск по номеру заказа находит все позиции одного заказа

## Продажа товаров с маркировкой (ЧЗ)

### Что происходит автоматически

- Товар перемещается с физического склада на виртуальный маркетплейса
- Статус меняется на **«Продан»**
- Заполняются данные для отчёта о выбытии (тип, причина, адрес, FIAS)

### Дедлайны

После продажи необходимо подать отчёт о выбытии в течение **3 рабочих дней**. На странице **«Сводка»** отображаются предупреждения:
- **Красный** — дедлайн просрочен
- **Жёлтый** — дедлайн через 1-2 дня

## Продажа товаров без маркировки

### Особенности

- Продаётся по **EAN-13** штрихкоду, артикулу или названию SKU
- **Без** отчёта о выбытии в ЧЗ
- Учёт только: дата продажи, номер заказа/чека, сумма
- Возврат на склад без ограничений
- Статус товара не проверяется (не требуется статус «В обороте»)

## Поиск товара

В нижнем блоке **«Поиск товара»** можно найти любую единицу по коду:
- Введите КМ, EAN-13 или артикул
- Нажмите **«Найти»** или ждите автопоиск (при вводе в поле добавления)
- Результат отображается с полной информацией о единице

Для очистки результатов нажмите **«Очистить»**.
