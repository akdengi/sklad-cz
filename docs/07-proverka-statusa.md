# 7. Проверка статуса маркировки товаров

## Что такое статус маркировки

Каждый код маркировки имеет статус в системе Честный Знак:

| Статус | Описание | Что значит |
|--------|----------|------------|
| **EMITTED** | Эмитирован | Код выпущен системой ЧЗ |
| **APPLIED** | Нанесён | Код нанесён на товар |
| **INTRODUCED** | В обороте | Товар введён в оборот |
| **RETIRED** | Выбыл | Товар выбыл из оборота |
| **WITHDRAWN** | Выведен | Товар выведен из оборота |
| **WRITTEN_OFF** | Списан | Товар списан |

## Проверка одного товара

### Через карточку единицы

1. Перейдите во вкладку **«Остатки»**
2. Найдите нужную единицу
3. Нажмите **«Изменить»**
4. Рядом с полем «Код маркировки ЧЗ» нажмите **«Проверить»**
5. Статус обновится автоматически

### Через настройки

1. Перейдите во вкладку **«Настройка»**
2. В секции **«Честный Знак — статусы»** нажмите **«Обновить статусы всех товаров»**
3. Дождитесь завершения (коды проверяются пакетами по 900)

## Массовая проверка

### Через интерфейс

1. Перейдите во вкладку **«Настройка»**
2. Нажмите **«Обновить статусы всех товаров»**
3. Система проверит все коды ЧЗ в базе
4. Результаты отобразятся в статусе операции

### Через API

```bash
# Запуск проверки
POST /api/cz/check-all

# Проверка статуса
GET /api/cz/check-status
```

## Требования для проверки

Для проверки статусов необходима настроенная авторизация:
- Сертификат ЭЦП в хранилище
- ИНН организации
- PIN-код токена

## Автоматическое обновление

При проверке статуса:
- Токен кэшируется и переиспользуется
- При ошибке 401 токен автоматически обновляется
- Статусы обновляются только в сторону увеличения (никогда не понижается)
- **Если код не найден** (ошибка 404 или другой код ошибки) — статус сбрасывается, отображается «КМ не найден в ЧЗ»
- При массовой проверке коды без ответа от ЧЗ также получают пустой статус

## Что делать при различных статусах

### EMITTED → APPLIED
Нужно нанести код на товар и подать отчёт о нанесении.

### APPLIED → INTRODUCED
Нужно подать отчёт о вводе в оборот через ЛК ЧЗ.

### INTRODUCED → RETIRED/WITHDRAWN
Товар выбыл из оборота. Подайте отчёт о выбытии через ЛК ЧЗ.

## Диагностика проблем

Если проверка не работает:
1. Перейдите в **«Настройка»** → **«Диагностика»**
2. Проверьте каждый шаг:
   - Сертификат найден?
   - PIN корректен?
   - Подпись формируется?
   - Авторизация проходит?
3. При ошибке «Подпись невалидна» — установите `csptestf` (входит в КриптоПро CSP)
4. При ошибке 403 «Отсутствует доступ» — добавьте сертификат как пользователь API на markirovka.crpt.ru
