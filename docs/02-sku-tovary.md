# 2. Создание SKU товаров и задание партий

## Что такое SKU

SKU (Stock Keeping Unit) — это карточка товара с основными параметрами: название, артикул, GTIN, партия. Каждый SKU может иметь несколько единиц товара.

## Типы товаров

Приложение поддерживает два типа товаров:

| Тип | Описание | Примеры |
|-----|----------|---------|
| **С маркировкой (ЧЗ)** | Товар с кодом маркировки Честный Знак | Лекарства, обувь, одежда с маркировкой |
| **Без маркировки** | Товар без кода ЧЗ, продаётся по EAN-13 | Продукты, household goods, non-regulated items |

## Создание нового товара

1. Перейдите во вкладку **«Товары»**
2. Нажмите **«Новый SKU»**
3. Заполните поля:

| Поле | Обязательно | Описание |
|------|-------------|----------|
| Название | Да | Наименование товара (например, «Футболка хлопковая») |
| **С маркировкой (ЧЗ)** | Да | Чекбокс. Если снят — товар без маркировки |
| Артикул | Нет | Внутренний артикул (например, «ФТБ-001») |
| Код ТН ВЭД | Нет | Код таможенной классификации. Нажмите «Найти» для поиска по справочнику |
| GTIN-14 | Да (только для ЧЗ) | 14-значный код товара. EAN-13 заполнится автоматически. Необязателен для товаров без маркировки |
| EAN-13 | Авто | 13-значный штрихкод (заполняется из GTIN, если тот начинается с 0) |
| Дата производства | Нет | Дата через календарь |
| Тираж | Нет | Максимальное количество единиц. Когда лимит достигнут, новые единицы создать нельзя |
| Разрешит. документация | Нет | Текст разрешительного документа |

4. Нажмите **«Сохранить»**

## Товары с маркировкой и без

### С маркировкой (ЧЗ)

- Стандартный поток: КМ → статусы → выбытие в ЧЗ
- Продажа через сканирование КМ кода
- Требуется отчёт о выбытии через ЛК ЧЗ
- Печать этикеток с DataMatrix

### Без маркировки

- Продажа по EAN-13 штрихкоду или артикулу
- **Без** отчёта о выбытии в ЧЗ
- Учёт: дата продажи, номер заказа/чека, сумма
- Возврат на склад без ограничений
- Печать этикеток без DataMatrix (текст + EAN-13)

## Редактирование и удаление

- В таблице товаров нажмите **«Изменить»** для редактирования
- **«Удалить»** — удалит SKU и все связанные единицы товара (с подтверждением)

## Тираж (лимит количества)

Поле **«Тираж»** задаёт максимальное количество единиц товара для данного SKU:
- При попытке создать больше единиц, чем тираж, система выдаст ошибку
- Это работает при массовом создании, сканировании и импорте
- Установите 0 или оставьте пустым для无限ного количества

## Прогресс маркировки

В таблице товаров отображается:
- **Бейдж** — «ЧЗ» для товаров с маркировкой, «Без ЧЗ» для товаров без маркировки
- **Прогресс** — для товаров с маркировкой: количество промаркированных / тираж
- **Количество** — для товаров без маркировки: общее количество единиц
