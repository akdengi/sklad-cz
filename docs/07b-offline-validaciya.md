# 7б. Оффлайн-валидация кодов маркировки

## Что такое оффлайн-валидация

Оффлайн-валидация — это проверка структуры кода маркировки (КМ) **без обращения к API Честного Знака**. Проверяется формат и содержимое кода.

## Что проверяется

### Структура кода (обязательные элементы)

| Элемент | AI | Формат | Описание |
|---------|-----|--------|----------|
| GTIN | 01 | 14 цифр | Global Trade Item Number |
| Серийный номер | 21 | 1–20 символов | Уникальный номер единицы |
| Ключ проверки | 91 | 4 символа | Ключ аутентификации (необязательно) |
| Криптохвост | 92 | 44 символа Base64 | Цифровая подпись (необязательно) |

### Проверки

1. **Наличие AI 01** — код должен начинаться с GTIN
2. **Длина GTIN** — ровно 14 цифр
3. **Наличие AI 21** — серийный номер обязателен
4. **Длина серийного номера** — рекомендуется 13 символов (стандартный тип)
5. **Допустимые символы** — латиница, цифры, спецсимволы
6. **Длина ключа (AI 91)** — 4 символа (если присутствует)
7. **Длина криптохвоста (AI 92)** — 44 символа (если присутствует)
8. **Допустимые символы в криптохвосте** — Base64: a-z, A-Z, 0-9, +, /, =

## Как запустить

### Через интерфейс

1. Перейдите во вкладку **«Настройка»**
2. В секции **«Честный Знак — статусы»** нажмите **«Оффлайн-валидация всех кодов»**
3. Система проверит все коды ЧЗ на остатках
4. Результаты отобразятся:
   - Если все коды валидны — зелёное сообщение
   - Если есть ошибки — модальное окно со списком проблемных кодов

### Автоматически

При открытии вкладки **«Остатки»** система автоматически проверяет единицы, у которых ещё нет сохранённого результата валидации, и кеширует его в базу.

### Через API

```bash
# Массовая валидация всех кодов на остатках
GET /api/units/validate-all

# Валидация одного кода
GET /api/units/validate?cz=<код>&sku_id=<id>
```

Ответ `validate-all`:
```json
{
  "total": 150,
  "invalid_count": 3,
  "invalid": [
    {
      "unit_id": 42,
      "sku_name": "Товар А",
      "cz_code": "010460000000000021...",
      "warnings": ["Серийный номер (AI 21): ожидается 13 символов, получено 14"]
    }
  ]
}
```

## Визуальная индикация в Остатках

Строки с проблемными кодами выделяются фоном:

| Цвет фона | Значение |
|-----------|----------|
| **Розовый** (`#fce4ec`) | Оффлайн-валидация не пройдена (ошибка структуры кода) |
| **Оранжевый** (`#fff3e0`) | Проблема со статусом в ЧЗ (BLOCKED / UNDEFINED / EMPTY) |
| **Жёлтый** | Нет кода ЧЗ |
| **Красный** | Товар возвращён |

Также:
- В колонке «Статус» появляется значок-предупреждение
- Код ЧЗ в ячейке подсвечивается розовым фоном при ошибке валидации
- При наведении на строку — всплывающая подсказка с причиной

## Онлайн-проверка и сброс статуса

При нажатии кнопки **«Проверить статус ЧЗ»** в карточке единицы:

- Если код найден в ЧЗ — статус обновляется
- Если код **не найден** (ошибка 404 или другой код ошибки) — статус сбрасывается, отображается **«КМ не найден в ЧЗ»**
- Массовая проверка («Обновить статусы всех товаров») также сбрасывает статус для кодов, не получивших ответ от ЧЗ

## Когда запускать

- После массового импорта кодов из CSV/TXT
- После ручного ввода кодов
- При подозрении на ошибки в кодах
- Регулярно для контроля качества данных

## Отличие от онлайн-проверки

| | Оффлайн-валидация | Онлайн-проверка |
|---|---|---|
| **Что проверяет** | Структуру кода | Статус в системе ЧЗ |
| **Обращение к API** | Нет | Да (требуется сертификат ЭЦП) |
| **Скорость** | Мгновенно | Медленно (сетевые запросы) |
| **Информация** | Формат кода | Текущий статус (эмитирован, в обороте и т.д.) |

## Сохранение результата

Результат оффлайн-валидации сохраняется в поле `cz_offline_valid` единицы:
- `true` — код прошёл валидацию
- `false` — код не прошёл валидацию
- `null` — валидация ещё не проводилась

Поле обновляется при:
- Запуске массовой оффлайн-валидации
- Создании/изменении единицы с кодом ЧЗ
- Импорте кодов из CSV/TXT/PDF
- Первом открытии вкладки «Остатки» (для единиц без кеша)
